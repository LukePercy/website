▶ Thinking  ·  exploring blog.ts, ISR flags, date parsing
  • Read src/lib/blog.ts — found isPublishedAtOrBeforeNow() filtering on Date.now()
  • Checked src/app/blog/page.tsx — no revalidate export — fully static
  • Checked all MDX frontmatter dates — all >= 2026-03-25
  • Last build timestamp 21:24 UTC March 24 — before UTC midnight Mar 25
  • Date-only ISO strings parse as UTC midnight — timezone edge case identified
AI
Root cause: Both blog pages are fully static — Date.now() is evaluated at build time, not per-request. The last build ran at 21:24 UTC on March 24, before UTC midnight March 25, so the post's publishedAt: "2026-03-25" was still in the future and got permanently excluded.

Fix: Add export const revalidate = 3600 to both blog pages. ISR re-runs the date filter hourly.
